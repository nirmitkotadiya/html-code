Types of paths

  • Absolute path — full URL or absolute path from root: /images/pic.jpg
  • Relative path — relative to the current file: images/pic.jpg or ../assets/pic.jpg

Examples

Link to another page in the same folder:

<a href="day5-forms.html">Go to Day 5</a>

Link to a file in a subfolder:

<img src="images/photo.png" alt="Photo">

Go up one folder:

<a href="../index.html">Up one level</a>

Path Examples Explained

Current File Target File Path to Use Explanation
index.html about.html about.html Same folder - just use filename
index.html logo.png images/logo.png File in subfolder - path to file from current location
pages/contact.html logo.png ../images/logo.png Go up one level (to parent folder) then into images folder
pages/contact.html index.html ../index.html Go up one level to parent folder

Absolute vs Relative Paths

Path Type When to Use Examples
Absolute Linking to external resources or resources at the root of your domain https://example.com/images/logo.png
/assets/css/style.css
Relative Linking to resources within your own website images/photo.jpg
../scripts/app.js

Special Path Symbols

  • ./ - Current directory (optional in most cases)
  • ../ - Parent directory (go up one level)
  • ../../ - Go up two levels
  • / - Root directory of the website

Tip

When testing locally, open index.html in your browser. For consistency when deploying, keep a clean folder structure (e.g., css/, images/, pages/).

Common Mistakes to Avoid

  • Using absolute paths to your local machine (e.g., C:/Users/Name/website/images/logo.png)
  • Incorrect number of ../ when navigating through folders
  • Case sensitivity issues (Linux servers are case-sensitive)
  • Forgetting to include file extensions
