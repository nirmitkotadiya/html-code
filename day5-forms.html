Basic form structure

The <form> element creates a container for form elements. The action attribute specifies where to send the form data, and the method attribute specifies how to send it (GET or POST). 

<form action="/submit" method="post">
  <!-- Form elements go here -->
</form>

Form Attributes

Attribute Description Example
required Makes the field mandatory <input type="text" required>
placeholder Hint text that disappears when user types <input placeholder="Enter name">
disabled Makes the field non-interactive <input type="text" disabled>
readonly Makes the field viewable but not editable <input type="text" readonly>
min/max Sets minimum/maximum values for number/range inputs <input type="number" min="1" max="10">
pattern Specifies a regex pattern for validation <input pattern="[A-Za-z]{3}">

Form Best Practices

  • Always use <label> elements for accessibility
  • Group related form controls with <fieldset> and <legend>
  • Use appropriate input types for better mobile experience
  • Validate inputs on both client and server side
  • Provide clear error messages when validation fails
  • Keep forms simple and only ask for necessary information
